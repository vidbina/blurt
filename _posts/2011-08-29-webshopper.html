A Few Thinks To Know Before You Read On

I've just landed a web dev gig that pays six-hundred big ones for a small web-shop meaning that it is about time to delve into the world of e-commerce. Being a big advocate of the open-source movement within my social circles, I naturally started on a quest to discover the available options on the market. At the moment I'm stationed in the Netherlands and therefore it seemed like the most natural move to pay some extra attention to the popular payment methods (read: iDEAL and Paypal) we are familiar with on this side of the planet. So here goes...

In the Background

For the try-outs I've just decided to run everything on a VirtualBoxed Ubuntu Server running LAMP and OpenSSH. Before I opted for this great solution I started trying things out on NearlyFreeSpeech.net but they have a pro safe mode policy that doesn't really sit too well with some web-shop systems. If your web-pages are stationed on a shared-hosting service, there is a large possibility that safe mode has been turned on by you admins. That brings me to the following point: be mindful to verify whether your host allows running websites with the PHP safe mode disabled. That would save you a lot of headache. I've read about the good and the bad, broken packages and well written systems and so many other reason to stick with the safe mode but in the end even the developers of the web-shop systems recommend you to turn the safe mode off. Who am I not to listen...

Open-Source Web-shops

There is a diverse set of flavors and colors to keep any web-developer entertained. The most interesting ones are, of course, the open-source solutions and because I'm Dutch, the iDEAL-friendly sweets are even more appealing. There is a list of web-shop platforms which have iDEAL plug-ins/modules/extensions that work well with iDEAL -- good place to start.

OpenCart

[caption id="" align="alignleft" width="440" caption="The slick-looking OpenCart webshop"][/caption]

The OpenCart system looked extremely slick at a first glance. Besides its obvious good looks the OpenCart system seemed to be quite easy to set-up. A bit of copy-paste work, a few chmod's to set access rights and a install procedure that was pretty self-explanatory. Once configured, managing the OpenCart shop was a pleasant walk in the park. Layouts, extensions, products - all easily set to your heart's desire from the panel but there is a a thing though... OpenCart won't function on hosts that don't enable their users to disable the SAFE MODE setting for PHP. This would not always be a problem, but for some hosts that calls for a lot of hassle.
